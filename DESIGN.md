# AI Gateway - 设计思考

## 气质与意象

一台精密的交换机柜——深灰色金属面板上排列着发光的端口指示灯，数据流在透明光纤中穿梭，安静而高效。没有多余的装饰，只有状态、方向、速度。这是基础设施的美学：可靠、透明、不插手内容只负责送达。

## 视觉策略

- 图像方向：深色背景上的数据流动可视化，端口连接状态指示
- 图形语言：极简的节点-连线拓扑图，实心圆点代表 Provider，带箭头的线代表活跃连接
- 整体感觉：开发者工具、终端美学、暗色系控制面板

## 配色方案

- 主色：翠绿 `#10b981`（emerald-500）— 端口活跃/连接成功的信号灯
- 辅色：琥珀 `#f59e0b`（amber-500）— 等待/警告状态
- 危险色：玫红 `#ef4444`（red-500）— 断开/错误
- 背景：深灰 `#0f172a`（slate-900）— 服务器机房暗色
- 卡片：`#1e293b`（slate-800）— 设备面板
- 文字：`#e2e8f0`（slate-200）— 仪表盘文字
- 次要文字：`#94a3b8`（slate-400）— 标签/说明

## 字体排版

- 中文：系统默认 -apple-system
- 英文/代码：monospace 优先（JetBrains Mono 风格）
- 排版节奏：紧凑，信息密度高，像终端输出

## 动效与交互

- 连接状态变化：指示灯 pulse 动画（1s 缓动）
- 请求转发：微妙的流动指示（非必要，性能优先）
- 切换 Provider：瞬间切换，无过渡延迟

## 页面结构

- 顶部：网关状态栏（运行中/停止、活跃连接数、当前代理端口）
- 主体左侧：Provider 列表（Agnes、DeepSeek 等卡片，显示状态和模型数）
- 主体右侧：当前活跃路由映射（Claude Code → DeepSeek, Codex → Agnes）
- 底部：实时请求日志（精简，最近 20 条）

## 设计禁忌

- 不要用渐变背景——这是工具不是营销页
- 不要用大面积留白——信息密度要高
- 不要用动画延迟交互——操作必须即时响应
- 不要用浅色主题——暗色是开发者工具的共识
